## 1. 概述

### 1.1 什么是工作流？

工作流是一系列有序的任务节点，定义了 Agent 团队如何协作完成复杂任务。每个节点可以：

- 绑定特定的 Agent 角色
- 配置执行参数
- 设置依赖关系
- 定义触发条件

### 1.2 工作流的核心概念

| 概念 | 说明 |
|------|------|
| **节点 (Node)** | 工作流中的单个任务单元 |
| **连接 (Edge)** | 节点之间的执行顺序关系 |
| **绑定 Agent** | 指定执行该节点的 Agent 角色 |
| **条件** | 决定是否执行或跳过的规则 |
| **触发器** | 启动工作流执行的事件 |

### 1.3 工作流类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **顺序工作流** | 节点按顺序依次执行 | 简单的线性流程 |
| **并行工作流** | 多个节点同时执行 | 独立任务并行处理 |
| **条件工作流** | 根据条件分支执行 | 需要判断的场景 |
| **混合工作流** | 结合多种类型 | 复杂业务流程 |

#### 连接节点

**方法一：拖拽连接**
1. 将鼠标移到源节点的底部连接点
2. 按住鼠标左键拖拽
3. 释放到目标节点的顶部连接点

**方法二：工具栏连接**
1. 选中源节点
2. 点击工具栏的 "🔗 连接" 按钮
3. 点击目标节点

#### 删除节点

1. 点击选中要删除的节点
2. 按 `Delete` 键或点击工具栏的 "🗑️ 删除" 按钮
3. 确认删除操作

> **注意**：删除节点会同时删除与之相关的所有连接。

### 3.3 参数配置

为节点配置执行参数。

```yaml
# 节点参数配置示例
node:
  id: "content-creation"
  name: "内容创作"
  assignedAgent: "writer"

  # 执行参数
  parameters:
    content_type: "article"
    word_count: 2000
    tone: "professional"
    keywords: ["AI", "Agent", "自动化"]

  # 超时设置
  timeout: 60  # 分钟

  # 重试配置
  retry:
    max_attempts: 3
    interval: 5  # 秒
```

- **筛选**：按关系类型筛选显示

## 6. 最佳实践

### 6.1 工作流设计原则

#### 1. 单一职责

每个节点应该只负责一个明确的任务：

```
❌ 不推荐：
节点: 内容生产和发布
- 写文章
- 设计图片
- 发布内容

✅ 推荐：
节点1: 内容撰写 (写手)
节点2: 图片设计 (设计师)
节点3: 内容发布 (运营)
```

#### 2. 合理粒度

节点粒度适中，不要过大或过小：

```
❌ 过大：节点执行时间 > 2小时
✅ 推荐：节点执行时间 15-60 分钟
❌ 过小：节点执行时间 < 5 分钟
```

#### 3. 明确依赖

清晰定义节点之间的依赖关系：

```yaml
# 显式声明依赖
dependencies:
  - source: "market-research"
    target: "content-creation"
    type: "data"
    description: "调研报告作为内容创作输入"
```

### 6.2 Agent 绑定建议

| 任务类型 | 推荐 Agent | 原因 |
|----------|------------|------|
| 数据分析 | 分析师 | 专业分析能力 |
| 内容创作 | 写手 | 文字表达能力 |
| UI 设计 | 设计师 | 视觉设计专长 |
| 代码开发 | 开发者 | 技术实现能力 |
| 项目协调 | 队长 | 管理协调能力 |

## 7. 常见问题

### Q1: 工作流执行卡住不动

**可能原因**：
1. Agent 离线或忙碌
2. 前置节点未完成
3. 条件不满足

**解决方案**：
1. 检查 Agent 状态，确保在线
2. 查看执行监控面板，定位卡住的节点
3. 检查条件表达式是否正确

### Q2: Agent 执行失败

**可能原因**：
1. 缺少必要技能
2. 参数配置错误
3. 资源不足

**解决方案**：
1. 检查节点技能配置，确保 Agent 已安装
2. 查看执行日志，定位具体错误
3. 尝试手动重试节点

### Q3: 条件分支不正确

**可能原因**：
1. 条件表达式语法错误
2. 变量引用不正确
3. 条件逻辑有问题

**解决方案**：
1. 检查条件表达式语法
2. 使用调试模式验证变量值
3. 简化条件逻辑，逐步测试

### Q4: 协作关系图显示异常

**可能原因**：
1. 节点依赖配置错误
2. 浏览器缓存问题

**解决方案**：
1. 检查节点依赖关系配置
2. 清除浏览器缓存并刷新
3. 重新加载工作流

### Q5: 工作流保存失败

**可能原因**：
1. 网络连接问题
2. 配置格式错误
3. 存储空间不足

**解决方案**：
1. 检查网络连接
2. 导出配置文件，检查格式
3. 清理旧的备份文件
