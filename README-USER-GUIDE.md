## 1. 快速开始（5 分钟上手）

### 1.1 安装步骤

#### 前置要求

- **Node.js** 18.0 或更高版本
- **npm** 或 **pnpm** 包管理器
- **OpenClaw CLI** 已安装并配置
- 运行中的 OpenClaw Gateway

> **Windows 用户**：本项目完全支持 Windows 10/11。请先安装 [Node.js](https://nodejs.org/) 和 [Git for Windows](https://git-scm.com/download/win)，然后使用 PowerShell 或 CMD 执行以下命令。

#### 安装命令

```bash
# 1. 克隆项目
git clone https://github.com/dhb861832993-star/clawteam.git
cd clawteam

# 2. 安装依赖
npm install

# 3. 自动配置环境（检测 OpenClaw 安装路径）
node scripts/detect-openclaw.js

# 4. 启动 Gateway（如果未运行）
openclaw gateway start

# 5. 启动 Dashboard
npm run dev:all
```

### 1.2 启动应用

启动成功后，打开浏览器访问：

```
http://localhost:5173
```

### 1.3 首次配置

首次启动后，建议完成以下配置：

1. **检查 Gateway 连接**：确认右上角显示 "SYSTEM ONLINE"
2. **创建团队**：点击 Header 中的 "Teams" 按钮创建您的第一个团队
3. **安装必要技能**：点击 "Skills" 按钮，从 ClawHub 市场安装需要的技能
4. **配置健康检查**：进入 Settings > Health Check，启用心跳监控

#### Agent 状态说明

| 状态 | 颜色 | 说明 |
|------|------|------|
| `ONLINE` | 🟢 绿色 | Agent 在线且空闲，可接收任务 |
| `WORKING` | 🟠 橙色 | Agent 正在执行任务 |
| `WAITING` | 🔵 蓝色 | Agent 等待输入或依赖 |
| `OFFLINE` | ⚪ 灰色 | Agent 离线 |
| `ERROR` | 🔴 红色 | Agent 发生错误 |

#### 操作方式

1. **查看 Agent 详情**：点击 Agent 卡片，右侧面板将显示详细信息
2. **选择多个 Agent**：按住 `Shift` 键点击可多选
3. **刷新状态**：状态会自动轮询更新，也可手动刷新页面

#### 安装新技能

1. 切换到 **ClawHub Market** 标签
2. 浏览或搜索需要的技能
3. 点击 **Install** 按钮安装

```bash
# 示例：安装 GitHub 技能
点击 "GitHub 操作" 卡片 → 点击 "Install" → 等待安装完成
```

#### 启用/禁用技能

- 点击技能卡片右下角的开关切换启用状态
- 禁用的技能不会影响 Agent 运行，但相关功能将不可用

#### 卸载技能

- 点击 "Uninstall" 按钮卸载
- 内置技能（builtin）无法卸载

## 6. 常见问题（FAQ）

### Q1: 无法连接 OpenClaw Gateway

**症状**：右上角显示红色状态，Agent 列表为空

**解决方案**：

1. 检查 Gateway 是否运行：
```bash
openclaw gateway status
```

2. 如果未运行，启动 Gateway：
```bash
openclaw gateway start
```

3. 检查 Gateway URL 配置：
```bash
# 查看当前配置
echo $OPENCLAW_GATEWAY_URL

# 默认值
export OPENCLAW_GATEWAY_URL=ws://127.0.0.1:18789
```

4. 检查防火墙设置，确保端口 18789 未被阻止

### Q2: Agent 显示离线但实际在运行

**症状**：Agent 卡片显示 OFFLINE，但 Agent 进程正常运行

**解决方案**：

1. 检查心跳配置：
```
Settings > Health Check > Enable Heartbeat: ON
```

2. 增加超时时间：
```
Settings > Health Check > Timeout: 15s (默认 10s)
```

3. 检查 Agent 工作空间路径是否正确：
```
Agent Card > Properties > workspace
```

### Q3: 配置不保存

**症状**：修改设置后刷新页面丢失

**解决方案**：

1. 确保点击了保存按钮：
```
Settings > Save Changes
```

2. 检查备份是否启用：
```
Settings > Backup > Enable Backup: ON
```

3. 手动导出配置：
```
Settings > Backup > Export Config
```

4. 检查浏览器本地存储是否被清除

### Q4: 技能安装失败

**症状**：点击 Install 后显示错误

**解决方案**：

1. 检查网络连接

2. 检查 ClawHub 市场是否可访问：
```bash
curl -I https://clawhub.openclaw.dev
```

3. 查看详细错误日志：
```
Agent Card > Logs > Filter: ERROR
```

4. 尝试手动安装：
```bash
openclaw skill install <skill-name>
```

### Q5: 任务执行卡住不动

**症状**：任务进度条长时间无变化

**解决方案**：

1. 检查依赖任务是否完成（DAG View）

2. 检查 Agent 是否在线：
```
Health Check Panel
```

3. 查看日志排查问题：
```
Right Panel > Logs
```

4. 尝试取消并重新创建任务

### Q6: 团队切换后任务丢失

**症状**：切换团队后看不到之前的任务

**解决方案**：

任务与团队绑定，切换团队只会显示该团队的任务。解决方案：

1. 切换回原团队查看任务
2. 将任务迁移到新团队（复制任务详情后重新创建）

### Q7: 性能监控数据显示异常

**症状**：响应时间或成功率数据不准确

**解决方案**：

1. 等待数据刷新（每 30 秒更新一次）
2. 清除浏览器缓存
3. 重启 Dashboard 服务：
```bash
npm run dev:all
```

### Q8: 批量权限设置不生效

**症状**：设置权限后 Agent 仍无法执行操作

**解决方案**：

1. 确认已选中目标 Agent（高亮显示）
2. 检查权限是否正确应用：
```
Batch Permissions Panel > 查看权限表格
```
3. 刷新页面后重新设置
